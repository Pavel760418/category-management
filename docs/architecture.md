# Архитектура решения

## Роль системы

Система проектируется как category management BI-platform, сохраняющая управленческую логику исходного Excel-файла и переводящая её в масштабируемую Python/Streamlit архитектуру.

## Целевая модель

- Excel остаётся одним из выходных артефактов, а не единственным носителем логики.
- Центральным источником становится нормализованный датафрейм SKU (canonical English columns).
- Все аналитические представления являются производными от нормализованной модели данных.
- При неполных данных применяется graceful degradation.
- Пользовательский слой (шаблоны, UI, ошибки) полностью русскоязычный.

## Слои

### 1. UI Layer
Streamlit-страницы (`streamlit_app.py` + корневой `pages/`), визуальные карточки,
таблицы, графики, загрузка файлов, ручной маппинг.
Все подписи полей и шаблонов берутся из `display_name_ru`.

Точка входа для Streamlit Community Cloud — `streamlit_app.py` в корне репозитория.

### 2. Application Services
Маппинг, валидация, расчёты, экспорт, генерация шаблонов.

### 3. Domain Core
KPI-логика, статусная логика, CatMan-структура, роли категории, нормативы.

### 4. Config / Schema Layer
YAML-конфиги:
- `field_schema.yaml` — canonical field + RU display + aliases + descriptions;
- `templates_meta.yaml` — русские имена файлов/листов, required/optional, инструкции;
- KPI, роли категории, стили.

Принцип: **внешний русский интерфейс / внутренний технический слой**.

### 5. Export Layer
Формирование Excel-артефактов в стиле управленческого инструмента с русскими заголовками.

## Поток данных при загрузке

```text
Русский Excel-шаблон
  → read_template_excel (лист по русскому имени)
  → MappingService.auto_map (aliases_ru + aliases_en → canonical)
  → ValidationService (сообщения на русском)
  → CalculationService (работа только с canonical keys)
  → Dashboard / Excel export (снова русские display names)
```

## BI-логика

Решение сохраняет ключевую бизнес-цепочку:
продажи → валовая прибыль → маржа → запас → оборачиваемость → ассортимент → закупки → выкладка → LFL → управленческие выводы.

## Расширяемость локализации

Сейчас основной язык — русский. Архитектура schema-слоя допускает добавление
`display_name_en` / `description_xx` без изменения расчётного ядра.
