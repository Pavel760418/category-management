# Category Management BI

Production-like Python/Streamlit проект для перевода Excel-инструмента категорийного менеджмента в масштабируемую BI-систему без разрушения исходной управленческой логики.

## Назначение

Проект сохраняет бизнес-логику существующего Excel-файла и переносит её в архитектуру, пригодную для GitHub, дальнейшей разработки и деплоя в Streamlit. Решение ориентировано на три уровня пользователей:
- категорийный менеджер;
- коммерческий директор;
- CEO.

Система сохраняет ключевой принцип исходного файла: **один источник данных — несколько аналитических представлений**.

## Русификация шаблонов (важно)

Пользовательский слой полностью на русском языке. Технический слой остаётся на canonical English keys.

| Слой | Язык | Примеры |
|------|------|---------|
| User-facing (Excel-шаблоны, UI загрузки, маппинг, ошибки) | **русский** | `Шаблон_данных_SKU.xlsx`, лист «Данные SKU», колонка «Продажи, руб.» |
| Internal (расчёты, schema keys, Python) | английский | `sku_code`, `sales_value`, `shelf_meters` |

Источник правды:
- `app/config/field_schema.yaml` — canonical поля + `display_name_ru` + aliases + описания;
- `app/config/templates_meta.yaml` — русские имена файлов/листов, обязательные/опциональные поля, инструкции.

Автосопоставление понимает **русские названия как основной вариант** и сохраняет английские синонимы для устойчивости импорта.

## Что внутри

- Streamlit-приложение в современном BI-стиле;
- модуль загрузки и маппинга Excel-файлов;
- graceful degradation при неполных данных;
- ядро расчётов KPI категории;
- экспорт итогового Excel-файла в стиле управленческого инструмента;
- **русскоязычные** шаблоны Excel для загрузки данных;
- конфиги KPI, ролей категории, цветов и metadata полей;
- тесты маппинга русских колонок и генерации шаблонов.

## Архитектура

### Бизнес-слои

1. **Ingestion layer** — загрузка Excel-файлов, чтение русских листов, очистка.
2. **Mapping layer** — автоматический и ручной маппинг колонок (RU → canonical).
3. **Validation layer** — проверка обязательных полей с русскими сообщениями.
4. **Calculation layer** — KPI: оборот, валовая прибыль, маржа, дни запаса, оборачиваемость, SKU, LFL, выкладка, ABC, бюджет закупок.
5. **Presentation layer** — Streamlit-дашборд и Excel-экспорт.
6. **Quality & Status layer** — отчёт по качеству данных и ограничениям анализа.
7. **Schema / Locale layer** — единый metadata-слой для display names и шаблонов.

### Принципы устойчивости

- приложение не падает при отсутствии файла;
- расчёты строятся по доступным данным;
- недоступные блоки помечаются явно;
- пользователь получает отчёт о качестве данных и ограничениях анализа;
- поддерживается ручной маппинг колонок;
- логика не зависит от отображаемого русского названия — только от canonical schema.

## Поддерживаемые загрузки (user-facing)

### 1. Шаблон_данных_SKU.xlsx
Лист: **Данные SKU**

Обязательные поля:
- Артикул SKU
- Наименование товара
- Категория
- Подкатегория
- Продажи, руб.
- Продажи, шт.
- Себестоимость, руб.
- Остаток, шт.
- Цена закупки, руб.

Необязательные поля:
- Длина выкладки, м
- Продажи LFL, руб.
- Продажи LFL, шт.
- Валовая прибыль LFL, руб.
- Магазин
- Формат магазина
- Период, мес.
- Роль SKU

### 2. Шаблон_конкуренты.xlsx
Лист: **Конкуренты**

Обязательные: Конкурент, Категория, Ценовой индекс.

### 3. Шаблон_цели_категории.xlsx
Лист: **Цели категории**

Обязательные: Показатель, Цель на 3 мес., Цель на 6 мес.

В колонке «Показатель» можно указывать русские названия KPI (Оборот, Дни запаса, …) или внутренние коды (`turnover`, `stock_days`).

## Деплой на Streamlit Community Cloud

Причина типовой ошибки `File does not exist: streamlit_app.py`: Cloud по умолчанию
ожидает файл **`streamlit_app.py` в корне репозитория**. Точка входа приложения —
именно он (не `app/main.py`).

Настройки приложения на share.streamlit.io:
- **Repository**: этот GitHub-репозиторий
- **Branch**: `main` (или ветка с актуальным кодом)
- **Main file path**: `streamlit_app.py`

Multipage-страницы находятся в корневом каталоге `pages/` — Streamlit подхватывает
их автоматически рядом с `streamlit_app.py`.
