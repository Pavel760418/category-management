# Category Management BI

Production-like Python/Streamlit проект для перевода Excel-инструмента категорийного менеджмента в масштабируемую BI-систему без разрушения исходной управленческой логики.

## Назначение

Проект сохраняет бизнес-логику существующего Excel-файла и переносит её в архитектуру, пригодную для GitHub, дальнейшей разработки и деплоя в Streamlit. Решение ориентировано на три уровня пользователей:
- категорийный менеджер;
- коммерческий директор;
- CEO.

Система сохраняет ключевой принцип исходного файла: **один источник данных — несколько аналитических представлений**.

## Что внутри

- Streamlit-приложение в современном BI-стиле;
- модуль загрузки и маппинга Excel-файлов;
- graceful degradation при неполных данных;
- ядро расчётов KPI категории;
- экспорт итогового Excel-файла в стиле управленческого инструмента;
- шаблоны Excel для загрузки данных;
- конфиги KPI, ролей категории, цветов и маппинга колонок;
- тестовый каркас для дальнейшего развития.

## Архитектура

### Бизнес-слои

1. **Ingestion layer** — загрузка Excel-файлов, чтение листов, очистка, унификация названий колонок.
2. **Mapping layer** — автоматический и ручной маппинг колонок, поддержка альтернативных названий.
3. **Validation layer** — проверка обязательных полей, типов данных, пустых строк, дублей и ошибок.
4. **Calculation layer** — расчёт KPI: оборот, валовая прибыль, маржа, дни запаса, оборачиваемость, SKU, LFL, эффективность выкладки, ABC, бюджет закупок.
5. **Presentation layer** — Streamlit-дашборд и Excel-экспорт в управленческом стиле.
6. **Quality & Status layer** — отчёт по качеству данных, статус загрузки файлов, ограничения анализа.

### Принципы устойчивости

- приложение не падает при отсутствии файла;
- расчёты строятся по доступным данным;
- недоступные блоки помечаются явно;
- пользователь получает отчёт о качестве данных и ограничениях анализа;
- поддерживается ручной маппинг колонок.

## Поддерживаемые загрузки

### 1. SKU base template
Основной шаблон загрузки данных категории. Обязательные поля:
- sku_code
- sku_name
- category
- subcategory
- sales_value
- sales_qty
- cogs_value
- stock_qty
- purchase_price

Опциональные поля:
- shelf_meters
- prior_sales_value
- prior_sales_qty
- prior_gp_value
- store_name
- format_name
- period_months
- current_role

### 2. Competitor template
Шаблон конкурентного мониторинга. Обязательные поля:
- competitor_name
- category
- price_index

Опциональные поля:
- width_sku
- depth
- shelf_meters
- stm_share
- promo_share
- comments

### 3. Goals template
Шаблон целевых показателей. Обязательные поля:
- metric_name
- target_3m
- target_6m

## Логика экранов приложения

### Главная
Описание решения, supported templates, статус текущей сессии загрузки.

### Загрузка данных
- загрузка Excel-файлов;
- выбор шаблона;
- автосопоставление колонок;
- ручной маппинг;
- отчёт о загруженных/незагруженных файлах.

### Качество данных
- пропуски;
- дубликаты;
- некорректные типы;
- пустые строки;
- ограничения аналитики.

### Dashboard
Главный управленческий экран в логике листа «🏆 Защита категории».

### Excel Export
Формирование итогового Excel-файла со следующими листами:
- Защита категории
- Данные SKU
- ABC-анализ
- Оборачиваемость
- План закупок
- Выкладка
- LFL и цели
- Конкуренты
- Статус данных
- Качество данных
- Ограничения анализа
- Инструкция

## Развитие

Следующие шаги расширения:
- расширение CatMan workflow: роль → оценка → цели → стратегия → тактики → review;
- сохранение ручных маппингов между сессиями через БД или S3;
- подключение мастер-данных по магазинам/форматам;
- richer benchmarking по конкурентам;
- ролевая модель пользователей.
